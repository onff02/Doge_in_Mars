# 🚀 Doge City in Mars  
**우주 항해로 재해석한 판단·심리 기반 의사결정 게임**

---

## 1. 프로젝트 개요

**Doge City in Mars**는 실제 시장의 변동성과 불확실성을  
**우주 항해 시뮬레이션**으로 재해석한 가로형 모바일 게임입니다.

플레이어는 인류 최후의 개척선 **도지(Doge)호**의 항해사가 되어,  
혼란스러운 정보 속에서 판단을 내리며 화성에 안전하게 도착해야 합니다.

이 게임은 **주식 게임이 아닙니다.**  
**판단과 심리를 다루는 게임**입니다.

---

## 2. 핵심 컨셉

- 📈 시장 신호 → **중력파 / 우주 환경**
- 🧠 투자 판단 → **엔진 레버 선택 (UP / DOWN)**
- 💥 리스크 → **연료 소모 / 선체 손상**
- 🪐 장기 성과 → **화성 착륙 및 도지 시티 규모**

정답보다 중요한 것은  
**불완전한 정보 속에서 어떤 판단을 반복했는가**입니다.

---

## 3. 게임 플레이 루프

1. **게임 시작**
   - 인트로 영상 재생
   - 우주선 탑승

2. **로켓 선택**
   - 각 로켓은 서로 다른 성향을 가짐
     - 성장형
     - 우량형
     - 방어형

3. **조종석 화면 (메인 플레이)**
   - 좌측: 중력파 안정도 차트
   - 우측: 인텔리전스 업데이트 (혼란·상충 정보)
   - 중앙: 엔진 레버 (UP / DOWN)

4. **결정**
   - 출력 증가(UP) 또는 감소(DOWN) 선택

5. **결과 반영**
   - 연료 및 선체 상태 갱신
   - 다음 라운드 진입

6. **최종 착륙**
   - 플레이 결과에 따라 화성 도지 시티 규모 결정

7. **AI 분석**
   - 플레이어의 판단 성향 분석 리포트 제공

---

## 4. 라운드 구성

게임은 총 **6개의 라운드**로 구성됩니다.

| 라운드 | 우주 환경 |
|------:|----------|
| 1 | 시스템 붕괴 구역 (Systemic Crash) |
| 2 | 고변동성 지대 (High Volatility) |
| 3 | 확장 항로 (Expansion) |
| 4 | 규제 성운 (Regulatory Fog) |
| 5 | 거품 붕괴 구역 (Bubble Collapse) |
| 6 | 최종 접근 단계 (Final Approach) |

각 라운드마다 **5개의 상충된 인텔 업데이트**가 제공되며,  
플레이어를 의도적으로 혼란에 빠뜨립니다.

---

## 5. 의사결정 시스템

각 라운드에서 플레이어는 하나의 선택을 합니다.

- **UP**: 공격적 가속
- **DOWN**: 방어적 안정화

결과는 다음 요소에 의해 결정됩니다.
- 현재 우주 환경
- 선택한 로켓의 특성
- 판단의 일관성

모든 정보는 불완전하며,  
**완벽한 선택은 존재하지 않습니다.**

---

## 6. AI 분석 시스템

게임 종료 후, **AI 항해 분석관**이 플레이를 평가합니다.

### AI가 분석하는 요소
- 혼란 정보에 대한 반응
- 리스크 선호도
- 낙관·비관 신호에 대한 민감도
- 환경 변화에 따른 판단 일관성

### AI가 하지 않는 것
- ❌ 현실 투자 조언
- ❌ 실제 기업/종목 언급
- ❌ 단순 정답/오답 판정

### AI 출력 결과
- 판단 성향 아키타입
- 강점 / 약점
- 한 줄 요약
- 학습 인사이트
- 종합 등급 (S–F)

## 7. 기술 스택

### 프론트엔드
- **Expo (React Native)**
- TypeScript
- Zustand (상태 관리)
- Skia (차트 렌더링)
- Expo AV (영상 재생)

### 백엔드
- Node.js (Express)
- REST API
- AWS EC2 배포

### AI
- OpenAI API (서버 사이드 전용)
- 커스텀 프롬프트 + 데이터 스키마
- 클라이언트 직접 호출 없음

---

## 8. 개발 참고 사항

- 게임 로직은 결정론적이지만, 정보는 의도적으로 왜곡됨
- AI 분석은 **게임 내부 데이터만 사용**
- 개발 단계에서는 Mock Game Session 사용 가능
- 플레이어에게 실제 금융 데이터는 노출되지 않음
